Plot notes: the text length of 'spam' is concentrated on higher lengths, while 'ham' is generally below 50 characters. This is more evident in the 2nd plot, where I used independent density normalization. It looks like there is a good chance that text length will help us classify between spam and ham.

Note that word Frequency will be addressed after pre-processing. This is because we're using words to help classify instead of generate sentiment. It makes less sense to remove a particular word that might be used a lot in either label, since that would reduce the amount of information available to models. Typically I'd generate word frequency here to use as a baseline for measuring how much cleaning accomplished(and what needs to be manually cleaned), but since I'm not interesting in removing more words than standard cleaning already does, I'm electing to defer word frequency until after cleaning.

def identify_tokens(df):
    row = df['text']
    tokens = nltk.word_tokenize(row)
    return tokens

df['tokens'] = df.apply(identify_tokens, axis=1)

def lemmatize_text(text):
    lemmatizer = nltk.WordNetLemmatizer()
    return [lemmatizer.lemmatize(w) for w in text]

def word_frequency(df):
    words = df.sum()
    count = Counter(words)
    count = pd.DataFrame.from_records(count.most_common(), columns=['word','count'])
    count['word'] = count['word'].str.lower()
    count = count.head(15)
    print(count)
