Train/Test Split¶

In Supervised ML we usually split data into 2 subsets: train and test data. We fit/train a model on train and make predictions on the test dataset. We then evaluate the prediction performance.

  • The training set contains a known output, where the model learns on this data to be able to generalized to other data later on.
  • The test dataset (or subset) is used to test our model’s prediction on this subset

We’ll do this using the Scikit-Learn library and specifically the train_test_split methods.

Train/test split method does have some limitations. — what if the split we make isn’t random?

  • What if one subset of our data has only people from a certain state, employees with a certain income level but not other income levels, only women or only people at a certain age? This will result in overfitting.

  • In order to avoid this, we can perform cross validation. (See discussion later)

Overfitting vs Underfitting¶

  • Model Underfitting: where a model can neither model the training data nor generalize to new data. An underfit ML model is obvious as it will have poor performance on the training data.

    • Underfitting is easy to detect given a good performance metric.
    • The solution is to try alternate ML algorithms

  • Model Overfitting : where a model learns the noise in the training data as to negatively impact model performance on new data. The noise or random fluctuations in the training data is learned as concepts by the model, however these concepts do not apply to new data (eg: the model can't generalize to new/unseen data)

    • more likely with nonparametric and nonlinear models that have more flexibility when learning a target function
    • many nonparametric ML algorithms also include parameters or techniques to limit/constrain how much detail the model learns

Both impact the predictability of a model, but by far the most common problem is overfitting. An optimun model is one at the sweet spot between underfitting and overfitting, but this is difficult to do in practice.

We can look at the performance of a ML algorithm over time as it is learning on training data.

  • We can plot both the performance on the training data and test dataset
  • Over time, as the algorithm learns, the error for the model on the training goes down and so does the error on the test dataset
  • If we train for too long, the performance on train may continue to decrease because the model is overfitting. At the same time the error for test starts to rise again as the model’s ability to generalize decreases
  • The sweet spot is the point just before the error on the test dataset starts to increase, where the model has good skill on both the training dataset and the unseen test dataset

There are 2 main techniques when evaluating ML algorithms to limit overfitting:¶

  • resampling to estimate model accuracy: the most popular is k-fold cross validation which allows you to train and test a model (k-times) on different subsets of training data and build up an estimate of its performance on unseen data.

  • hold back/validation dataset: a subset of your training data held back from your ML algorithms, used to evalaute model performance. After you have selected and tuned your ML algorithms on the training dataset, you can evaluate the learned model(s) on the validation dataset, to get an idea of how it might perform on unseen data.

One approach to limiting overfitting is by varying a key model hyperparameter and evaluating the model performance on the train/test sets for each configuration using k-fold validation

k-Fold Cross-Validation¶

Cross validation is very similar to train/test split, but applied to more subsets. We split our data into k subsets, and train on k-1 one of those subset, while holdiing the last subset for test. There are various cross validation methods: K-Folds cross-validation, Leave One Out cross-validation. We will discuss k-fold.

The k-fold cross-validation procedure divides a limited dataset into k non-overlapping folds. Each of the k folds is given an opportunity to be used as a held back test set, while all other folds collectively are used as a training dataset. A total of k models are fit and evaluated on the k hold-out test sets and the mean performance is reported. A value of 5 or 10 folds is typical.

  • StratifiedKFold takes the cross validation a step further, where the class distribution in the dataset is preserved in the training and test splits. Therefore, in classifications tasks with imbalanced class distributions, we should prefer StratifiedKFold over KFold

  • cross_val_score: takes the dataset and applies cross validation to split the data. Then, train a model using the specified algorithm (e.g. logistic regression, decision tree, …) and measure the performance of the model (scoring parameter). The procedure is as follows:

    • The model is trained using K-1 of the folds as training data
    • The resulting model is validated on the remaining data
    • This process is repeated K times and performance measure such as "ACCURACY" is computed at each step.

Machine Learning Algorithms¶

Supervised machine learning¶

In supervised learning algorithms using known datasets (eg: training datasets) to make predictions. These algorithms learn the mapping function from input variables (x) to an output variable (y); eg: y = f(X). The objective is to approximate the mapping function (f) as accurately as possible such that whenever there is new input data (x), the output variable (y) for the dataset can be predicted.

Regression and classification are examples of supervised machine learning. Their main difference is that y in Regression is continuous (numerical) while for classification, it is categorical (or discrete).

Regression algorithms estimate the mapping function (f) from the input variables (x) to numerical or continuous output variables (y). For example, given a dataset about houses, a regression task is to predict housing prices where price is a continuous output. Examples of the common regression algorithms include

  • linear regression
  • Support Vector Regression (SVR) and regression trees.

Classification in machine learning estimate the mapping function (f) from the input variables (x) to discrete or categorical output variables (y). When provided with one or more input variables, a classification model attempts to predict the value of a single or several conclusions. For example: predicting whether prices for houses “sell more or less than the recommended retail price.” Here, houses are classified whether their prices fall into two discrete categories: "above" or "below" this price. Examples of the common classification algorithms include:

  • logistic regression: predicting a binary outcome (eg: whether a tumor is cancerous or not)
  • Naïve Bayes
  • decision trees
  • KMeans (K Nearest Neighbors) (note: an unsupervised ML algorithm)

Unsupervised Machine Learning¶

Unsupervised models are used when the class label of each sample is not available. Unsupervised clustering methods such as k-means create groups with instances that have similarities. If one does not have the class labels associated with dataset, one can use clustering methods to help identy classes by finding out related instances. Then use supervised methods can be used to learn from new labeled data set. Examples of unsupervised algorithms:

  • Hierarchical clustering
  • K-means clustering
  • K-NN (k nearest neighbors)
  • Principal Component Analysis (for feature reduction)

Assessing the Fit of Regression Models¶

A well-fitting regression model results in predicted values close to the observed data values. 3 statistics are used in Ordinary Least Squares (OLS) regression to evaluate model fit:

  • R-squared: ranges from 0 to 1, with 0 indicating the model does not improve prediction, and 1, perfect prediction. Improvement in the regression model results in a proportional increases in R-squared. A pitfall of R-squared is that it increases as predictors are added to the model. This increase is artificial when predictors are not actually improving the model’s fit. One remedy is the related statistic, Adjusted R-squared that incorporates the model’s degrees of freedom. Adjusted R-squared decreases as predictors are added if the increase in model fit does not make up for the loss of degrees of freedom and will increase as predictors are added, only if the increase in model fit is worthwhile. Adjusted R-squared should always be used with models with more than one predictor variable and is interpreted as the proportion of total variance that is explained by the model. When the interest is in the relationship between variables, not in prediction, the R-square is less important.

  • F-test: evaluates the null hypothesis that all regression coefficients are equal to 0 vs the alternative that at least one is not. An equivalent null hypothesis is that R-squared equals 0. A significant F-test indicates the observed R-squared is reliable and not a spurious result of oddities in the data. Thus the F-test determines whether the proposed relationship between the response variable and the set of predictors is statistically reliable and can be useful when the research objective is either prediction or explanation.

  • Root Mean Square Error (RMSE): is the square root of the variance of the residuals. It indicates the absolute fit of the model to the data–how close the observed data points are to the model’s predicted values. RMSE can be interpreted as the standard deviation of the unexplained variance, and has the useful property of being in the same units as the response variable. Lower values of RMSE indicate better fit. RMSE is a good measure of how accurately the model predicts the response, and it is the most important criterion for fit if the main purpose of the model is prediction.

Decision Tree¶

Decision tree is a systematic approach for multiclass classification. It poses a set of questions to the dataset (related to its attributes/features). The decision tree classification algorithm can be visualized on a binary tree. On the root and each of the internal nodes, a question is posed and the data on that node is further split into separate records that have different characteristics. The leaves of the tree refer to the classes in which the dataset is split.

We can use grid search for hyper parameter tuning. For decision treens, hyperparameters include:

  • criterion (gini or entrop, default=gini)
  • splitter (best or random, default=best)
  • max_depth (default is None).
  • In Scikit-learn, optimization of decision tree classifier performed pre-pruning. Maximum depth of the tree can be used as a control variable for pre-pruning

    #use gridsearch to test all values
