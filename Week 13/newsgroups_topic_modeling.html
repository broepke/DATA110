Topic Modelling¶

Topic Modeling is a process to automatically identify topics from large collections of text, for discovery of hidden patterns exhibited by a text corpus. It is used for finding groups of words (eg: topic) from a collection of documents that best represents the information in the collection. It has application in the following:

  • Discovering hidden topical patterns that are present across the collection
  • Annotating documents according to these topics
  • Using these annotations to organize, search and summarize texts

Topic modeling can be implemented using cluster analysis such as K-means, LDA (Latent Dirichlet Allocation), as well as other techniques. In this session we look at cluster analysis and topic models. So what's the difference between these two methods?

In general, if a document is assigned to only a single topic, this is document clustering. However, if each document can have more than one topic, this task relates to the decomposition methods, such as LDA.

  • Both are unsupervised tasks that attempts to organize/group documents
  • Text clustering looks for similarity among documents to form similar document clusters. The similarity measures could be based on TF-IDF weighting.
  • Topic models don't look at document similarity, but instead, treats a document as a mixture of topics in which a topic is a probability distribution of words, such that each document is assigned a probability of belonging to a latent theme or topic.
  • Another advantage between topic models and conventional cluster analysis, is they use more sophisticated iterative Bayesian techniques to determine the probability that each document is associated with a given topic. So while the documents are initially given a random probability of being assigned to topics, these probabilities become increasingly accurate as more data are processed.

def clean_message(doc):
    
    doc = doc.lower()                                                         # lower-case 
    doc = re.sub(r"http\S+", '', doc)                                         # remove urls
    doc = re.sub(r"nntp\S+", '', doc)                                         # remove urls
   
    doc = re.sub(r'\S*@\S*\s?', '', doc)                                      # remove email
    doc = re.sub(r'\d+', '', doc)                                             # remove numbers
    doc = re.sub('/\r?\n|\r/', '', doc)                                      # remove new line
    doc = re.sub("\'", '', doc)                                              # Remove single quotes
    doc = ' '.join(lemma.lemmatize(word) for word in doc.split())              # lemmatization
    doc = ' '.join([w for w in word_tokenize(doc) if w not in useless_words])  # stop, punctuation removal  
    
    return doc.split()


def getCleanText(doc):
    clean_lines = []
    for line in doc:
        line = line.strip()   # remove leading/trailing spaces
        cleaned = clean_message(line)
        cleaned = ' '.join(cleaned)
        clean_lines.append(cleaned)
    
    return clean_lines

K-Means Model¶

Clustering is a common data analysis technique used to get an intuition about the structure of the data. The goal is to find homogeneous subgroups within the data such that data points in each cluster are as similar as possible according to a similarity measure such as euclidean-based distance or correlation-based distance.

Unlike supervised learning, clustering is an unsupervised learning method since we dont have the ground truth to compare the output of the algorithm to the true labels to evaluate its performance. Since kmeans requires k as an input and doesnt learn it from data, there is no right answer in terms of the number of clusters needed for a problem and often domain knowledge and intuition may help. However there are 2 metrics that may provide some intuition about k:

  • Elbow method: provides an idea on a good k number of clusters based on the sum of squared distance (SSE) between data points and their assigned clusters centroids
  • Silhouette analysis: used to determine the degree of separation between cluster. The coefficient values range from (-1, 1) where 0 indicates the sample is very close to the neighboring cluster, 1 indicates the sample is far away from the neighboring cluster and -1 indicates the sample is assigned to the wrong clusters. Hence, we want coefficients to be as big as possible and close to 1 to have a good clusters.

Latent Dirichlet Allocation (LDA)¶

LDA is a topic model used to classify text in a document to a particular topic. It builds a topic per document model and words per topic model, modeled as Dirichlet distributions. LDA assumes documents are produced from a mixt of topics found within the entire corpus. Those topics then generate words based on their probability distribution. Note: the topic structure is hidden (or latent) as we only observe the documents/words, not the topics themselves. Therefore this method is used to infer topic structure, given the known words and documents.

The basic approach starts with preparing a document-term matrix. For each document, count how many times a particular term occurs. Since not all terms are equally important, TF-IDF weighting can be used rather than raw counts. The next step is to decompose this matrix into document-topic and term-topic matrices. Next, the analyst will look at the main terms of each topic in order to label or identify the names of these topics. Since number of topics is far fewer than the vocabulary, topic modelling can also be used as a dimensionality reduction technique.

The basic pipeline/process involves:

  • preparing documents
  • cleaning/preprocessing
  • preparing document term matrix
  • running LDA model
  • results evaluation
    • human judgment/Observation-based (eg. observing the top N words in a topic)
    • quantitative metrics such as perplexity (held out likelihood) and coherence calculations
    • mixture of human judgement and quantitative

LDA topic modeling with Gensim¶

In the following, we will use Gensim (a free open-source Python library) for LDA topic modeling. The core concepts of gensim are:

  1. Document: anything from a short tweet, a paragraph (i.e., journal article abstract), a news article, or a book

  2. Corpus: a collection of documents. Corpora serve two roles in Gensim:

    • Input for training a Model: During training, a model uses this training corpus to look for common themes/topics.

    • Documents to organize: After training, a topic model can be used to extract topics from new documents (eg: unseen in the training corpus).

After collecting our corpus, there are typically a number of preprocessing steps such as removing high frequency words (eg: stop words), stemming/lemmatization, etc..

  1. Vector: a mathematically convenient representation of a document.

  2. Model: an algorithm for transforming vectors from one representation to another.

Bag of Words¶

To infer the latent structure in our corpus we need a way to represent documents that we can manipulate mathematically. One approach to represent a document as a vector is the bag-of-words model. Here each document is represented by a vector containing the frequency counts of each word in the dictionary.

For example: assuming a dictionary containing the words ['coffee', 'milk', 'sugar', 'spoon'].

A document consisting of the string "coffee milk coffee" would be represented by the vector [2, 1, 0, 0]

Building the Topic Model¶

In addition to the corpus and dictionary, we need to provide the number of topics to build a topic model.

  • alpha and eta are hyperparameters affecting sparsity of the topics. According to the Gensim docs, both defaults to 1.0/num_topics prior.

  • chunksize: the number of documents to be used in each training chunk

  • update_every: determines how often the model parameters should be updated
  • passes: the total number of training passes

Model Perplexity and Coherence Score¶

Model perplexity and topic coherence provide a convenient measure to judge how good a given topic model is.

In [79]:
def lda_scoring (model, corpus, text, dictionary):

    # Compute Perplexity
    print('\nPerplexity: ', model.log_perplexity(corpus))  # a measure of how good the model is. lower the better.

    # Compute Coherence Score
    coherence_model_lda = CoherenceModel(model=model, 
                                         texts=text, 
                                         dictionary=dictionary, 
                                         coherence='c_v')
    
    coherence_lda = coherence_model_lda.get_coherence()
    print('\nCoherence Score: ', coherence_lda)
